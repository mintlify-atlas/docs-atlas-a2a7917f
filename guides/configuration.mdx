## Overview

The Asistencias system uses a centralized configuration file (`db.php`) that handles database connections, character encoding, and input sanitization functions. This guide covers all configuration aspects.

## Database Connection

The main configuration file is located at the root of the project:

<CodeGroup>
```php db.php
<?php
$host = "localhost";
$user = "root";
$password = "";
$db = "asistencia_practicantes";

$conn = new mysqli($host, $user, $password, $db);

if ($conn->connect_error) {
    die("Connection failed: " . $conn->connect_error);
}

$conn->set_charset("utf8mb4");

function sanitize_input($data) {
    $data = trim($data);
    $data = stripslashes($data);
    $data = htmlspecialchars($data);
 
    return $data;
}
?>
```
</CodeGroup>

## Configuration Parameters

<Steps>
  <Step title="Database Host">
    Set the MySQL server hostname:

    ```php
    $host = "localhost";
    ```

    **Options:**
    - `localhost` - For local development
    - `127.0.0.1` - IP address alternative
    - Remote server IP or hostname for production

    <Note>
      For production deployments, use the actual database server hostname or IP address.
    </Note>
  </Step>

  <Step title="Database User">
    Configure the MySQL user with appropriate permissions:

    ```php
    $user = "root";
    ```

    <Warning>
      Never use the `root` user in production! Create a dedicated database user with limited privileges:
      
      ```sql
      CREATE USER 'asistencias_user'@'localhost' IDENTIFIED BY 'strong_password';
      GRANT SELECT, INSERT, UPDATE, DELETE ON asistencia_practicantes.* TO 'asistencias_user'@'localhost';
      FLUSH PRIVILEGES;
      ```
    </Warning>
  </Step>

  <Step title="Database Password">
    Set the password for your database user:

    ```php
    $password = "";
    ```

    <Warning>
      The default empty password is only suitable for local development. Always use strong passwords in production environments.
    </Warning>
  </Step>

  <Step title="Database Name">
    Specify the database name (must match the one created during setup):

    ```php
    $db = "asistencia_practicantes";
    ```

    <Info>
      This should match the database name from your [Database Setup](/guides/database-setup) process.
    </Info>
  </Step>
</Steps>

## Connection Management

### MySQLi Connection

The system uses MySQLi (MySQL Improved Extension) for database operations:

```php
$conn = new mysqli($host, $user, $password, $db);
```

### Error Handling

Connection errors are caught and displayed:

```php
if ($conn->connect_error) {
    die("Connection failed: " . $conn->connect_error);
}
```

<Note>
  In production, replace `die()` with proper error logging to avoid exposing connection details to users.
</Note>

### Character Encoding

UTF-8 encoding is set for proper international character support:

```php
$conn->set_charset("utf8mb4");
```

**Why utf8mb4?**
- Supports full Unicode character set
- Handles emojis and special characters
- Compatible with international names and text

## Input Sanitization

The configuration file includes a global sanitization function used throughout the application:

```php
function sanitize_input($data) {
    $data = trim($data);           // Remove whitespace
    $data = stripslashes($data);   // Remove backslashes
    $data = htmlspecialchars($data); // Convert special characters
    
    return $data;
}
```

### Usage in Application Files

This function is used in authentication and registration:

<CodeGroup>
```php login.php (line 7)
$correo = sanitize_input($_POST['correo']);
```

```php registra_sesion.php (line 10)
$nombre = sanitize_input($_POST['nombre']);
```
</CodeGroup>

<Info>
  The `sanitize_input()` function is a first layer of defense but should always be combined with prepared statements for database queries. See [Security Guide](/guides/security) for more details.
</Info>

## Environment-Specific Configuration

### Development Environment

```php
<?php
// Development configuration
$host = "localhost";
$user = "root";
$password = "";
$db = "asistencia_practicantes";

error_reporting(E_ALL);
ini_set('display_errors', 1);
```

### Production Environment

```php
<?php
// Production configuration
$host = "production-db.example.com";
$user = "asistencias_user";
$password = getenv('DB_PASSWORD'); // Use environment variables
$db = "asistencia_practicantes";

error_reporting(E_ALL);
ini_set('display_errors', 0);
ini_set('log_errors', 1);
ini_set('error_log', '/var/log/php/errors.log');
```

<Warning>
  Never commit production credentials to version control. Use environment variables or secure configuration management tools.
</Warning>

## Session Configuration

Sessions are initialized in authentication files:

<CodeGroup>
```php login.php (line 3)
session_start();
```

```php registra_sesion.php (line 3)
session_start();
```
</CodeGroup>

### Session Variables

The following session variables are set upon successful login:

```php
$_SESSION['usuario_id'] = $usuario['id'];      // User ID
$_SESSION['nombre'] = $usuario['nombre'];      // User name
$_SESSION['rol_id'] = $usuario['rol_id'];      // Role ID (1=admin, 2=supervisor, 3=practicante)
```

### Session Security

For enhanced session security in production:

```php
session_start([
    'cookie_lifetime' => 0,
    'cookie_secure' => true,      // HTTPS only
    'cookie_httponly' => true,    // No JavaScript access
    'cookie_samesite' => 'Strict' // CSRF protection
]);
```

## File Structure

The configuration file is included at the top of all PHP files:

```php
require_once 'db.php';
```

**Files using db.php:**
- `login.php` - Authentication
- `registra_sesion.php` - User registration  
- `admin/guardar_usuario.php` - User management
- All dashboard and module files

<Info>
  The relative path may vary depending on directory structure. Admin files use `../db.php` to reference the configuration file.
</Info>

## Testing the Configuration

Create a test file to verify your database connection:

```php test_connection.php
<?php
require_once 'db.php';

if ($conn->ping()) {
    echo "Database connection successful!\n";
    echo "Database: $db\n";
    echo "Character set: " . $conn->character_set_name() . "\n";
} else {
    echo "Database connection failed: " . $conn->error;
}

$conn->close();
?>
```

Run from command line:
```bash
php test_connection.php
```

Expected output:
```
Database connection successful!
Database: asistencia_practicantes
Character set: utf8mb4
```

## Troubleshooting

### Common Connection Issues

<CodeGroup>
```text Access Denied
Connection failed: Access denied for user 'root'@'localhost'

Solution: Verify username and password are correct
```

```text Unknown Database
Connection failed: Unknown database 'asistencia_practicantes'

Solution: Run the database setup script (bd.sql)
```

```text Connection Refused
Connection failed: Can't connect to MySQL server

Solution: Ensure MySQL service is running
```
</CodeGroup>

### Verify MySQL Service

```bash
# Linux/Mac
sudo systemctl status mysql

# Or
sudo service mysql status

# Start if stopped
sudo systemctl start mysql
```
