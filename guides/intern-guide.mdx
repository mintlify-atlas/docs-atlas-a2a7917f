## Welcome, Practicantes!

This guide will help you use the Asistencias system to track your daily attendance at your municipal internship. The system makes it easy to check in when you arrive and check out when you leave.

<Info>
**Role ID:** 3 (Practicante)

**Your Capabilities:** Mark attendance, view attendance history, update profile
</Info>

## Daily Attendance Process

### Marking Your Entry

<Steps>
  <Step title="Arrive at Your Location">
    When you arrive at your internship site, open the Asistencias system on your device.
  </Step>
  
  <Step title="Log In">
    If not already logged in, enter your email and password.
  </Step>
  
  <Step title="Click Marcar Entrada">
    On your dashboard, click the **"Marcar Entrada"** button.
    
    ```php
    if (isset($_POST['marcar_entrada'])) {
        $query = "INSERT INTO asistencia 
                  (usuario_id, fecha, hora_entrada, hora_salida) 
                  VALUES ('$usuario_id', DATE(NOW()), TIME(NOW()), NULL)";
        mysqli_query($conn, $query);
        header('Location: practicante_dashboard.php');
        exit;
    }
    ```
  </Step>
  
  <Step title="Verify">
    The page will refresh and the button will change to **"Marcar Salida"**. Your entry time is now recorded.
  </Step>
</Steps>

<Tip>
The system automatically captures the current date and time when you click the button. You don't need to enter it manually.
</Tip>

### Marking Your Exit

<Steps>
  <Step title="Before Leaving">
    When you're ready to leave for the day, access your dashboard again.
  </Step>
  
  <Step title="Click Marcar Salida">
    Click the **"Marcar Salida"** button.
    
    ```php
    if (isset($_POST['marcar_salida'])) {
        $query = "UPDATE asistencia 
                  SET hora_salida = TIME(NOW()) 
                  WHERE usuario_id = '$usuario_id' 
                  AND fecha = DATE(NOW())";
        mysqli_query($conn, $query);
        header('Location: practicante_dashboard.php');
        exit;
    }
    ```
  </Step>
  
  <Step title="Confirm Exit">
    The page refreshes and your exit time is recorded. Tomorrow, the button will reset to **"Marcar Entrada"**.
  </Step>
</Steps>

<Warning>
You can only mark your exit AFTER you've marked your entry for the day. The system checks for today's attendance record before allowing exit marking.
</Warning>

$stmt = $conn->prepare("SELECT u.nombre, u.correo, u.password_hash,

$stmt->bind_param("i", $usuario_id);

### Editing Your Profile

Below the information display, you'll find an **"Editar Usuario"** section where you can update your profile.

<Steps>
  <Step title="Navigate to Profile Page">
    Click **"Perfil"** in the sidebar menu to access your profile.
  </Step>
  
  <Step title="Scroll to Edit Section">
    Below the "Información del Usuario" card, you'll see the "Editar Usuario" form.
  </Step>
  
  <Step title="Update Your Information">
    Modify any of the following fields:
    
    - **Nombre**: Change your display name
    - **Correo**: Update your email address (this changes your login email)
    - **Contraseña**: Enter a new password (optional - leave blank to keep current)
    
    ```php
    <form action="perfil.php" method="POST">
        <input type="text" name="nombre" value="<?php echo $usuario['nombre']; ?>">
        <input type="email" name="correo" value="<?php echo $usuario['correo']; ?>">
        <input type="password" name="password">
        <button type="submit" name="actualizar_perfil">Actualizar</button>
    </form>
    ```
  </Step>
  
  <Step title="Submit Your Changes">
    Click the **"Actualizar"** button to save your changes.
    
    The system processes your update securely:
    
    ```php
    if ($_SERVER['REQUEST_METHOD'] === 'POST') {
        $nombre = $_POST['nombre'];
        $correo = $_POST['correo'];
        $password = $_POST['password'];

        // Only update password if a new one is provided
        if (!empty($password)) {
            $password_hash = password_hash($password, PASSWORD_DEFAULT);
        } else {
            $password_hash = $usuario['password_hash'];
        }

        $update_query = "UPDATE usuarios 
                         SET nombre = ?, correo = ?, password_hash = ? 
                         WHERE id = ?";
        $update_stmt = $conn->prepare($update_query);
        $update_stmt->bind_param("sssi", $nombre, $correo, 
                                 $password_hash, $usuario_id);
        $update_stmt->execute();
    }
    ```
  </Step>
  
  <Step title="Verify Success">
    After submission, you'll see a success message: **"Perfil actualizado con éxito."**
    
    The information card at the top will display your updated details.
  </Step>
</Steps>

<Warning>
**Password Field Behavior**

- The password field is **optional** - leave it empty to keep your current password
- If you enter a password, it must be your new desired password
- The system automatically hashes passwords using secure bcrypt encryption
- You cannot view your current password (it's encrypted in the database)
</Warning>

<Tip>
**Important:** If you change your email address, remember to use the new email for your next login. The email field serves as your username in the system.
</Tip>

### Profile Update Security

The profile update feature includes several security measures:

<Accordion title="Password Hashing">
  Passwords are never stored in plain text. The system uses PHP's `password_hash()` with `PASSWORD_DEFAULT`:
  
  ```php
  $password_hash = password_hash($password, PASSWORD_DEFAULT);
  ```
  
  This uses bcrypt, one of the most secure password hashing algorithms.
</Accordion>

<Accordion title="Prepared Statements">
  All database updates use prepared statements to prevent SQL injection attacks:
  
  ```php
  $update_stmt = $conn->prepare("UPDATE usuarios 
                                  SET nombre = ?, correo = ?, password_hash = ? 
                                  WHERE id = ?");
  $update_stmt->bind_param("sssi", $nombre, $correo, $password_hash, $usuario_id);
  ```
  
  Your input is never directly inserted into SQL queries.
</Accordion>

<Accordion title="Session Validation">
  The profile page verifies your identity on every request:
  
  ```php
  session_start();
  if (!isset($_SESSION['usuario_id']) || $_SESSION['rol_id'] != 3) {
      header('Location: ../login.php');
      exit;
  }
  ```
  
  Only authenticated interns with `rol_id = 3` can access intern profiles.
</Accordion>

## Understanding Your Attendance Records

### Record Structure

Each attendance record contains:

<Tabs>
  <Tab title="Fecha (Date)">
    The date of your attendance in YYYY-MM-DD format.
    
    ```php
    fecha = DATE(NOW())  // Current date
    ```
  </Tab>
  
  <Tab title="Hora Entrada (Entry Time)">
    The exact time you clicked "Marcar Entrada" in HH:MM:SS format.
    
    ```php
    hora_entrada = TIME(NOW())  // Current time
    ```
  </Tab>
  
  <Tab title="Hora Salida (Exit Time)">
    The exact time you clicked "Marcar Salida" in HH:MM:SS format.
    
    ```php
    hora_salida = TIME(NOW())  // Current time
    ```
    
    Shows NULL or empty if you haven't marked exit yet.
  </Tab>
</Tabs>

### Database Storage

Your attendance is stored in the `asistencia` table:

```sql
CREATE TABLE asistencia (
    id INT AUTO_INCREMENT PRIMARY KEY,
    practicante_id INT NOT NULL,
    fecha DATE NOT NULL,
    hora_entrada TIME NOT NULL,
    hora_salida TIME NULL,
    FOREIGN KEY (practicante_id) REFERENCES practicantes(id)
);
```

## Common Scenarios

### Forgot to Mark Entry

<Warning>
Once the day has passed, you cannot retroactively mark attendance. Contact your supervisor if you forgot to check in.
</Warning>

### Forgot to Mark Exit

Your record will show as "Incompleta" (incomplete) to supervisors. The `hora_salida` field will be NULL. Mark it the next time you're on-site if possible, or inform your supervisor.

### Marked Wrong Time

You cannot edit attendance records through the UI. Contact your supervisor or administrator to correct mistakes.

### Can't See Today's Button

The system checks for today's attendance record:

```php
$query = "SELECT * FROM asistencia 
          WHERE usuario_id = '$usuario_id' 
          AND fecha = DATE(NOW())";
$result = mysqli_query($conn, $query);
$asistencia = mysqli_fetch_assoc($result);
```

If `$asistencia` exists, you'll see "Marcar Salida". Otherwise, "Marcar Entrada".

## Security and Access

### Authentication

All intern pages verify your identity:

```php
session_start();
if (!isset($_SESSION['usuario_id']) || $_SESSION['rol_id'] != 3) {
    error_log("Intento de acceso no autorizado. User ID: " . 
              ($_SESSION['usuario_id'] ?? 'Not set') . ", Role ID: " . 
              ($_SESSION['rol_id'] ?? 'Not set'));
    header('Location: ../login.php');
    exit;
}
```

<Info>
Unauthorized access attempts are logged. Only users with `rol_id = 3` can access intern features.
</Info>

### Password Security

Your password is hashed before storage:

```php
$password_hash = password_hash($password, PASSWORD_DEFAULT);
```

The system uses PHP's built-in password hashing with bcrypt algorithm for security.

## Troubleshooting

### Cannot Log In

<Steps>
  <Step title="Verify Credentials">
    Double-check your email and password. Email addresses are case-sensitive.
  </Step>
  
  <Step title="Check Role Assignment">
    Your account must have `rol_id = 3`. Contact your administrator if you're being redirected elsewhere.
  </Step>
  
  <Step title="Reset Password">
    Ask your supervisor to reset your password if you've forgotten it.
  </Step>
</Steps>

### Button Doesn't Appear

- Refresh the page
- Clear your browser cache
- Try a different browser
- Verify you're logged in with the correct account

### Attendance Not Showing

If you clicked the button but don't see the record:

1. Check the database connection status
2. Refresh the page to see if it appears
3. Contact your supervisor if the issue persists
