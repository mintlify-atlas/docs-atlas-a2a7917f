## Welcome to Asistencias

Asistencias is a comprehensive attendance tracking system designed for managing municipal interns (practicantes). This guide will help you get your first admin account created and start tracking attendance.

<Note>
  This quickstart assumes you have already completed the [installation](/installation) process and have a working MySQL database.
</Note>

## Getting Started

<Steps>
  <Step title="Verify Your Installation">
    Before creating your first account, ensure your web server is running and you can access the application. Navigate to your installation URL (e.g., `http://localhost/asistencias`).
    
    You should be automatically redirected to the login page.
    
    <Info>
      The application automatically redirects from `index.html` to `login.php` for all visitors.
    </Info>
  </Step>

  <Step title="Register Your First Admin Account">
    Since this is your first time using the system, you'll need to create an admin account.
    
    1. From the login page, click on **"¿No tienes una cuenta? Registrarse"** (Don't have an account? Register)
    2. You'll be taken to `registra_sesion.php`
    
    Fill out the registration form with the following information:
    
    - **Nombre Completo**: Your full name
    - **Correo Electrónico**: Your email address (must be valid format)
    - **Contraseña**: Choose a secure password
    - **Confirmar Contraseña**: Re-enter your password
    - **Rol**: Select **"Admin"** from the dropdown
    
    <Warning>
      Make sure to select the "Admin" role for your first account. This will give you full access to manage users, interns, and attendance records.
    </Warning>
    
    Click **"Registrarse"** to create your account.
  </Step>

  <Step title="Log In to Your Account">
    After successful registration, you'll see a success message: **"Registro exitoso. Puede iniciar sesión."**
    
    Click the **"Iniciar Sesión"** link or navigate back to `login.php` and enter:
    
    - Your registered email address
    - Your password
    
    Click **"Iniciar Sesión"** to access the system.
    
    <Info>
      The system uses secure password hashing with bcrypt. Your credentials are protected with industry-standard security practices.
    </Info>
  </Step>

  <Step title="Access the Admin Dashboard">
    Upon successful login, you'll be automatically redirected to the Admin Dashboard (`dasboards/admin_dashboard.php`) based on your role.
    
    The system has three role-based dashboards:
    - **Admin** (rol_id: 1) → Admin Dashboard
    - **Supervisor** (rol_id: 2) → Supervisor Dashboard
    - **Practicante** (rol_id: 3) → Practicante Dashboard
  </Step>
</Steps>

## How Authentication Works

The system implements secure authentication using PHP sessions and prepared statements:

```php login.php:10-24
// Prepare SQL to prevent SQL injection
$stmt = $conn->prepare("SELECT id, nombre, password_hash, rol_id FROM usuarios WHERE correo = ?");
$stmt->bind_param("s", $correo);
$stmt->execute();
$result = $stmt->get_result();

if ($result->num_rows === 1) {
    $usuario = $result->fetch_assoc();
    
    // Verify password
    if (password_verify($password, $usuario['password_hash'])) {
        // Start session
        $_SESSION['usuario_id'] = $usuario['id'];
        $_SESSION['nombre'] = $usuario['nombre'];
        $_SESSION['rol_id'] = $usuario['rol_id'];
```

Key security features:
- **Prepared statements** prevent SQL injection attacks
- **Password verification** uses PHP's `password_verify()` with bcrypt hashing
- **Session management** tracks authenticated users securely
- **Role-based redirects** ensure users land on appropriate dashboards

## Registration Process

Here's how the registration system validates and creates new users:

```php registra_sesion.php:33-38
// Encriptar la contraseña
$password_hash = password_hash($password, PASSWORD_BCRYPT);

// Preparar y ejecutar la consulta de inserción
$stmt = $conn->prepare("INSERT INTO usuarios (nombre, correo, password_hash, rol_id) VALUES (?, ?, ?, ?)");
$stmt->bind_param("sssi", $nombre, $correo, $password_hash, $rol);
```

The registration includes:
- **Input sanitization** using the `sanitize_input()` function
- **Email validation** with PHP's `FILTER_VALIDATE_EMAIL`
- **Password confirmation** to prevent typos
- **Duplicate checking** to ensure unique email addresses
- **Bcrypt hashing** for secure password storage

## Common Issues

<AccordionGroup>
  <Accordion title="I can't log in with my credentials">
    - Verify you're using the correct email address
    - Ensure your password is exactly as you registered it (passwords are case-sensitive)
    - Check that your user account was successfully created in the database
    - Verify the database connection is working in `db.php`
  </Accordion>
  
  <Accordion title="Registration shows 'El correo ya está registrado'">
    This email address is already in use. Either:
    - Use a different email address
    - Log in with the existing account
    - Contact an admin to remove the duplicate account
  </Accordion>
  
  <Accordion title="I get redirected to login.php immediately after logging in">
    This indicates a session issue. Check:
    - PHP sessions are enabled on your server
    - The session directory is writable
    - Your browser accepts cookies
    - There are no errors in the PHP error log
  </Accordion>
</AccordionGroup>

## Security Best Practices

<Warning>
  For production deployments, ensure you:
  - Change the default database credentials in `db.php`
  - Use HTTPS to encrypt all traffic
  - Implement proper file permissions (restrict write access)
  - Keep PHP and MySQL updated to the latest versions
  - Enable PHP error logging but disable error display
  - Implement rate limiting for login attempts
</Warning>

## Need Help?

If you encounter any issues during the quickstart:

1. Review the [Installation Guide](/installation) for setup requirements
2. Check the database connection in `db.php`
3. Verify all SQL tables were created correctly from `bd.sql`
4. Check your web server's PHP error logs for detailed error messages
